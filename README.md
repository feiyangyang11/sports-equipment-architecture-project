## 系统架构

本项目采用前后端分离的单体分层架构，后端主要遵循：

`controller -> service -> dao -> database`

其中：

- `controller`：提供 RESTful 接口入口，负责参数接收、返回结果和基础鉴权。
- `service`：承载业务规则，例如预约校验、状态流转、库存一致性控制。
- `dao`：负责访问 MySQL，执行查询、插入、更新等数据库操作。
- `database`：保存用户、器材、预约、借还等核心业务数据。

这种结构的目标是让接口层、业务层和数据访问层职责分离，便于维护、调试和后续扩展。

## 核心模块

当前版本主要包含以下模块：

- `auth`：登录、获取当前用户、退出登录。
- `equipment_category`：器材分类查询。
- `equipment`：器材分页查询、分类筛选、详情查询。
- `reservation`：学生预约申请、查询、取消；管理员审核通过与拒绝。
- `borrow_record`：管理员办理借出与归还；学生和管理员查询借用记录。
- `frontend pages`：学生端与管理员端的基础业务页面。

其中，`operation_log` 当前作为后续扩展项预留，暂未纳入第一版实现范围。

## 项目概述

项目名称：基于 C++ 的体育器材预约与借还管理系统设计与实现

项目目标：面向校园体育器材管理场景，设计一个支持器材查询、预约、审核、借出、归还、逾期跟踪和统计查询的中等规模软件系统。系统采用前后端分离思路，后端以 C++17 + Oat++ 实现核心业务逻辑，数据库采用 MySQL，前端采用 HTML + CSS + JavaScript + Bootstrap 完成基础交互。

## 技术栈

- 后端语言：`C++17`
- 后端框架：`Oat++`
- 构建工具：`CMake`
- 数据库：`MySQL / MariaDB`
- 前端：`HTML + CSS + JavaScript + Bootstrap 5`
- 本地运行辅助：`Python 3` 静态文件服务脚本、Windows `.cmd` 启动脚本
