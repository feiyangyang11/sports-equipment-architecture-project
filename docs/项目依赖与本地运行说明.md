# 项目依赖与本地运行说明

## 1. 当前运行环境

本项目当前按以下环境开发和验证：

- 操作系统：`Windows 10/11 x86_64`
- C++ 工具链：`MSYS2 UCRT64 + MinGW GCC`
- 构建工具：`CMake 3.20+`
- 数据库：`MySQL 8.0+` 或兼容的 `MariaDB`
- 前端运行方式：本地静态文件服务
- 浏览器：最新版 `Edge / Chrome`

说明：

- 当前仓库中的启动脚本主要面向 **Windows 本地开发**。
- 如果换成 Linux/macOS，可以继续使用这套源码结构，但脚本和路径需要自己适配。

---

## 2. 必装依赖总表

### 2.1 通用依赖

- `Git`
  用于克隆仓库。
- `Python 3`
  用于启动前端静态页，本项目不依赖 npm。
- `MySQL8.0` 
  用于保存业务数据。**这个去MySQL官网安装，建议跟着b站或者csdn其他教程安装，不然容易出点问题。**
- 现代浏览器
  用于访问前端页面。

### 2.2 后端编译依赖

后端使用 `C++17 + Oat++ + MariaDB/MySQL C Client`，因此需要：

- `MSYS2` ：去[MSYS2](https://www.msys2.org/)官网下载x86-64版本的，当然官网也有详细的下载安装教程
- `UCRT64` ：一个终端环境，下载MYSY2后基本会自带的，该终端就在msys64目录下。我的在`"C:\msys64\ucrt64.exe"`
- `g++`：可以在UCRT64终端中安装
- `cmake`：可以在UCRT64终端中安装，也可以去官网安装
- `mingw32-make`：可以在UCRT64终端中安装
- `oatpp`：可以在UCRT64终端中安装
- `libmariadbclient`：可以在UCRT64终端中安装

如果使用 MSYS2 UCRT64，建议安装这些包：

```bash
pacman -S --needed \
  mingw-w64-ucrt-x86_64-gcc \
  mingw-w64-ucrt-x86_64-cmake \
  mingw-w64-ucrt-x86_64-make \
  mingw-w64-ucrt-x86_64-oatpp \
  mingw-w64-ucrt-x86_64-libmariadbclient
```

说明：

- 本项目的后端 `CMakeLists.txt` 里显式依赖 `oatpp` 和 `mariadbclient`。
- 当前脚本默认按 `C:\msys64\ucrt64` 这套安装路径寻找运行时和库文件。

### 2.3 前端依赖

前端没有 `npm`、`package.json`、`node_modules` 依赖，属于**静态前端**。没有需要安装的东西

但当前页面使用了在线 CDN 资源：

- `Bootstrap 5.3.3`
- `Google Fonts`

因此如果本机完全断网，前端页面可能出现：

- 样式加载不完整
- 字体加载失败
- Bootstrap 弹窗脚本无法从 CDN 取到

所以当前前端除了本地文件本身，还依赖：

- `https://cdn.jsdelivr.net`
- `https://fonts.googleapis.com`
- `https://fonts.gstatic.com`

## 4. 数据库准备步骤

### 4.1 创建数据库

先创建数据库，在MySQL命令行执行：

```sql
CREATE DATABASE sports_equipment_management
CHARACTER SET utf8mb4
COLLATE utf8mb4_unicode_ci;
```

或者用数据库可视化软件直接建库

### 4.2 导入建表脚本

在MySQL命令行执行：

```bash
mysql -u root -p sports_equipment_management < database/schema/01_init.sql
```

或者用数据库可视化软件直接打开查询控制台执行该sql

### 4.3 导入初始化数据

在MySQL命令行执行：

```bash
mysql -u root -p sports_equipment_management < database/seed/01_seed.sql
```

或者用数据库可视化软件直接打开查询控制台执行该sql

### 4.4 修改数据库配置文件

后端读取：

- [database.conf](/D:/newdir/sports-equipment-architecture-project/backend/config/database.conf)

至少需要确认以下字段：

```ini
host=127.0.0.1
port=3306
database=sports_equipment_management
username=root
password=你的数据库密码
charset=utf8mb4
```

注意：

- 这里的 `username/password` 是你自己的**数据库连接账号密码**
- 不是系统前台登录页的用户密码

## 6. 后端构建与运行方式

### 6.1 一键脚本

仓库已经提供了 3 个 Windows 脚本，在项目根目录下的script目录中：

- [build-backend.cmd](/D:/newdir/sports-equipment-architecture-project/scripts/build-backend.cmd)
- [run-backend.cmd](/D:/newdir/sports-equipment-architecture-project/scripts/run-backend.cmd)
- [run-frontend.cmd](/D:/newdir/sports-equipment-architecture-project/scripts/run-frontend.cmd)

推荐顺序：

1. 双击 `build-backend.cmd`
2. 双击 `run-backend.cmd`
3. 双击 `run-frontend.cmd`

### 6.2 手动构建后端

如果你不想用脚本，也可以手动在UCRT64终端中执行：

```powershell
cd D:\newdir\sports-equipment-architecture-project\backend #这是我自己的项目目录
& cmake -S . -B build -G "MinGW Makefiles" -DCMAKE_PREFIX_PATH=C:/msys64/ucrt64
& "C:\Program Files\CMake\bin\cmake.exe" --build build
```

如果你和我一样在UCRT64终端中安装的cmake有点小问题，就像我一样执行下面的：

```powershell
cd D:\newdir\sports-equipment-architecture-project\backend #这是我自己的项目目录
& "C:\Program Files\CMake\bin\cmake.exe" -S . -B build -G "MinGW Makefiles" -DCMAKE_PREFIX_PATH=C:/msys64/ucrt64 #这是我自己的msys目录
& "C:\Program Files\CMake\bin\cmake.exe" --build build
```

**这里面的cmake可执行程序路径要改成你自己的**，否则也没法执行成功

### 6.3 手动运行后端

同理，下面的路径替换成你自己的

```powershell
cd D:\newdir\sports-equipment-architecture-project\backend
$env:PATH = "C:\msys64\ucrt64\bin;$env:PATH"
.\build\sports_equipment_backend.exe
```

后端默认监听：

```text
http://127.0.0.1:8000
```


## 8. 前后端联调前提

当前项目已经补好了最小 CORS，所以前端可以从：

- `http://127.0.0.1:5500`

访问后端：

- `http://127.0.0.1:8000`

也就是说，前后端联调的最小前提是：

1. 数据库已导入 schema 和 seed
2. `database.conf` 配置正确
3. 后端已启动
4. 前端静态页已启动

## 10. 克隆后本地运行的最短路径

别人拉取仓库以后，最短可复现路径如下：

1. 安装 `Git`
2. 安装 `MySQL` 或 `MariaDB`
3. 安装 `MSYS2 UCRT64`
4. 安装 `Python 3`
5. 在 `UCRT64` 中安装：
   - `gcc`
   - `cmake`
   - `make`
   - `oatpp`
   - `libmariadbclient`
6. 创建数据库 `sports_equipment_management`
7. 导入：
   - `database/schema/01_init.sql`
   - `database/seed/01_seed.sql`
8. 修改：
   - `backend/config/database.conf`
9. 运行：
   - `scripts/build-backend.cmd`
   - `scripts/run-backend.cmd`
   - `scripts/run-frontend.cmd`
10. 浏览器打开：
   - `http://127.0.0.1:5500/pages/login.html`

做到这一步，项目就可以在本地完整跑起来。
